## Issue #8 — Income Tax Returns ❌ 2 BUGS FOUND

### Bug #1: Wrong column name `invoice_date` → should be `issue_date`
**Files**: `api/src/Action/Report/IncomeTaxReturnFoAction.php:137-139`, `api/src/Action/Report/IncomeTaxReturnPoAction.php:130-132`

```php
// WRONG (column doesn't exist):
AND pi.invoice_date >= ?
AND pi.invoice_date <= ?

// CORRECT (per migration 0020):
AND pi.issue_date >= ?
AND pi.issue_date <= ?
```

The `purchase_invoices` table has `issue_date DATE NOT NULL` — there is no `invoice_date` column. These queries will fail at runtime.

### Bug #2: Incomplete invoice status filter in `getAnnualIncome()`
**Files**: `api/src/Action/Report/IncomeTaxReturnFoAction.php:114`, `api/src/Action/Report/IncomeTaxReturnPoAction.php:108`

```php
// WRONG — only counts 'issued', misses 'sent', 'reminded', 'paid'
AND i.status = 'issued'

// CORRECT (matches DphReportAction / InvoiceRepository::getVatSummary):
AND i.status IN ('issued', 'sent', 'reminded', 'paid')
```

Also `getAnnualExpenses()` only includes `('received', 'paid')` — should it also include `'booked'`? This is a business logic question (booked invoices may or may not count as deductible expenses).
